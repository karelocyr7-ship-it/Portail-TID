# Contribuer

1. Créer une branche dédiée à partir de `main`.
2. Décrire le besoin, les risques et le rollback.
3. Limiter le diff au besoin demandé.
4. Ajouter ou mettre à jour les tests et la documentation.
5. Exécuter `make check` avant la Pull Request.
6. Demander une revue humaine pour tout changement sensible.
